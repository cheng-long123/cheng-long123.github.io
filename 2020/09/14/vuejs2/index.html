VUE基础（二）

03-vue基础-计算属性

问题：模板内处理数据，如果逻辑较为复杂，对应的js表达式也会复杂，模板内的业务逻辑繁重，可维护性差

通过计算属性，来处理这些复杂逻辑，降级模板的复杂度，提供模板的可维护性。

使用场景：

  • 依赖data中的数据，但是需要经过复杂逻辑处理，才能得到你想要的数据。
  • 此时你可以使用计算属性，复杂逻辑在声明计算属性时候进行，模板内使用数据即可。

处理前：模板内逻辑复杂

处理后：模板内逻辑清晰 （使用 computed 配置选项）

  • computed 是vue的配置选项
  • 指向的一个对象，对象可声明多个计算属性
  • 属性名称（计算属性的名称）
  • 属性的值（计算属性的素材函数）
    • 对需要依赖的数据，进行逻辑上的处理
    • 处理完毕后，需要return出去，返回的值就是计算属性的值
  • 在模板中使用计算属性，和使用data的方式是一样的。
    • 虽然在计算属性中声明的是函数，但是在模板中使用，当中数据来使用，不需要加括号。

总结：

  • 当你依赖data中的数据，要经过较为复杂的逻辑处理，才能得到你想要的数据，此时可以使用计算属性。

04-computed和methods区别

methods定义函数，如果在模板中使用，每使用一次，就相当于调用了一次，处理逻辑会重新执行。

computed定义计算属性，如果在模板中使用，使用多次，但是如果依赖的数据不发生改变，计算属性对应的函数不会重新执行。

  • 计算属性会做缓存，提高渲染的性能。
  • 当依赖data中的数据，发生变化的时候，计算属性也会对应发生改变。

总结：

  • 计算属性有缓存，提高渲染性能。
  • 当依赖数据发生变化，计算属性也会发生改变。

06-工具-json-server

  • 快速创建接口的工具，模拟接口的工具。
  • 正常项目分工：后台提供接口服务，前端调用接口，实现前端业务逻辑。
  • 前后端分离
    • 两种分离模式：
      • 代码上的分离（前后端是同一个项目，开发目录上的区分）
      • 项目级的分离（前端是一个单独的项目，后台是一个单独的项目）
    • 我们的理解：必须后台先提供接口，前端才能进行开发。
    • 分离的意义：前后端可以并行开发。

json-server意义：模拟接口，推进前端开发进程。例外有一个工具：mock.js 模拟接口。

具体的使用步骤：

  1. 是一个基于nodejs命令行工具
  1. 准备一个空文件夹，任何目录都行

  1. 在文件夹中新建一个json文件，名称是英文。

  1. 在json文件中，定义一个对象，对象中会有 属性名 –> 属性值
    1. 属性名 接口的地址
    2. 属性值 接口返回数据 （数组|对象）
  1. 启动接口服务，在文件夹下，打开命令行窗口

  1. 测试，在浏览器访问上面地址

总结：注意下json格式。

Vue组件

01-★组件-概念

什么是组件？

  • js模块，独立的一个js文件，提供js逻辑，函数。
  • vue组件，关注于界面，它包含：HTML结构 CSS样式 JS代码，它可以把一个完整的页面，根据界面功能拆分成若干组件。

组件化开发好处？

  • 高度复用
  • 组件与组件之间相互独立的，例如：数据，函数。组件实现的功能足够单一，后期更好维护。

组件底层是什么？

  • 组件本质上是一个vue实例，但是写法和vue实例不同，new Vue({//配置对象})
  • 组件会通过 Vue.componnet(//组件) 或者在Vue实例中 new Vue({components:{//组件}})
  • 组件也有配置对象，不包含 el 选项，el是指定vue实例管理的视图容器，组件实通过 template选项，指定html结构容器的。
  • 配置选项：data methods watch filters directives computed created …（创建组件，创建vue实例）

05-★组件-命名规则

给大家梳理一下将来大家可能遇见的组件注册命名。

两种命名规范：

  • 注册组件组件名称：小写单词加中线 com-add nav-bar
    • 在使用组件的时候：<com-add></com-add> <nav-bar></nav-bar>
  • 注册组件组件名称：单词首字母大写 ComAdd NavBar
    • 在使用组件的时候：<com-add></com-add> <nav-bar></nav-bar>
    • 如果想直接使用 单词首字母大写 组件名称作为标签的名称
      • 只能template指定的视图中使用 <ComAdd></ComAdd> <NavBar></NavBar>
      • 通过el指定的dom容器中不能使用

总结：不管使用 小写单词加中线 还是 单词首字母大写 注册的组件，都使用 小写单词加中线 标签一定没错。

06-★组件-组件传值

由于组件的数据是相互独立的，导致某个组件需要使用另外一个组件的数据遇到问题，我们需要通过特定的方式来 进行组件间的数据通信。

然后这些不同的传值方式，和组件与组件间的关系有关。

分成三种：

  • 父组件 传值 子组件
  • 子组件 传值 父组件
  • 非父子关系，组件传值 （后面讲到）

父组件 传值 子组件：

  • 父组件 com-parent
  • 子组件 com-child

总结：父传子

  • 在父组件中，使用子组件的时候，通过自定义属性绑定父组件数据 :myMsg=”msg”
  • 在子组件中，使用props选项来接收父传递给子的数据：props:[‘myMsg’]

子组件 传值 父组件：

  • 父组件 com-parent
  • 子组件 com-child

推导：

  • 父组件 <span @click="fn1"> 是在给span标签绑定事件，指定的fn1函数是父组件的。
  • 如果我想触发fn1函数执行，触发span标签的点击事件。
  • 如果click绑定在span上就是原生事件，但是如果绑定在组件com-child上那就是自定义事件。
  • 那么 在组件上绑定的事件名称可以随意命名，它都是一个自定义事件。
  • 如果需要触发自定义事件，<com-child @myEvent="fn"> ，所以myEvent事件属于子组件触发。
  • vue实例提供了一个函数可以触发，自己组件绑定的事件：
    • this.$emit('myEevent') myEevent 自定义事件的名字
    • this.$emit('myEevent','数据') 事件对于的函数有一个默认的传参
      • 指定的函数没有括号 fn 拥有一个默认传参，就是触发事件的时候传递的数据
      • 指定的函数有了括号 fn($event) 这个$event就是就是触发事件的时候传递的数据。

07-SPA-特点介绍

当页面应用程序，简称SAP（single page application）,大白话：一个系统上的所有功能在一个页面上实现。

如果是所有功能在一个页面上实现，业务复杂度是非常高的，可维护性非常差。所以vue框架可以拆分业务，

使用组件来拆分业务。

优点：

  • 利于实现前后端分离而且是项目级的分离
  • 只有资源加载完毕后，场景与场景之间的切换很流畅。

缺点：

  • 它把所有功能写在一起，一个页面的资源提交较大，第一次访问这个系统很慢（首屏加载数据慢）
    • 这个后期可以做优化。
  • 在一个页面上实现，数据都是异步加载，js动态渲染，不利于SEO(搜索引擎优化)

在单页面应用程序中，如何切换页面的业务场景。

08-SPA-前端路由

后端路由：

  • 根据用户的请求地址+方式，分发到不同的业务函数，进行逻辑处理。
  • 控制 客户端请求 和 后端处理函数的 映射关系。

前端路由：

  • 根据地址栏变化（不发请求），去局部更新不同的页面内容。
    • 前端业务场景切换。
  • 在vue项目中的路由
    • 控制 地址变化 和 组件渲染 的映射关系。
    • 例如： / 对应渲染 首页组件
    • 例如：/login 对应渲染 登录组件

16-vue-cli-介绍

vue-cli是一个开发vue项目的脚手架工具。

脚手架是为了保证各施工过程顺利进行而搭设的工作平台。

vue-cli可以提供基于vue项目架子的快速创建，成为了一套标准。

  • 项目目录结构，统一的项目结构。
  • 提供了开发过程中的系列工具，
    • 例如：babel 语法降级的
    • 例如：eslint 约束语法风格（代码风格）
    • 例如：less 预处理器
    • …
  • 提供一个开发时服务器，预览代码（预览项目）
    • 提供：自动刷新浏览器，方便你预览
    • 提供：热更新功能，有些资源的修改，不需要刷新浏览器，立即更新，进行预览
    • …

总结：

  • vue-cli是一个提供开发项目过程中，便利的一个平台。更加方便的开发项目，提高你的开发效率。
  • 基于nodejs的命令行工具。

20-vue-cli-ES6模块化

模块化规范：用来约定模块（一个js文件就是一个模块，模块拥有自己作用域）的导入和导出。

在nodejs中，模块化规范：CommonJS

  • 导入：const path = require('path')
  • 导出：module.exports = '导出的内容' exports.xxx = '导出的内容'

ES6也提供了模块化规范：ES6模块

  • 默认导入与导出
    • 导入：import 变量名 from '包名|js模块路径'
    • 导出：export default 导出内容
  • 按需导入与导出
    • 导入：import {成员变量名1} from '包名|js模块路径'
    • 导出：
      • export const 成员变量名1 = '导出内容1'
      • export const 成员变量名2 = '导出内容2'
